# WeRead Eye Care 手工验收清单

## 1. 构建与加载

1. 在项目根目录运行：

```bash
npm install
npm test -- --run
npx tsc --noEmit
npm run build
```

2. 打开 `chrome://extensions`
3. 打开 `开发者模式`
4. 点击 `加载已解压的扩展程序`
5. 选择项目里的 `dist/`

验收标准：

- 扩展可以正常加载
- 没有 manifest 错误
- popup 可以打开
- options 页面可以打开
- popup 能看到当前模式和当前策略

## 2. 微信读书页面识别

1. 打开非微信读书页面，例如任意普通网页
2. 观察是否出现摄像头权限请求
3. 再打开微信读书阅读页 `https://weread.qq.com/web/reader/...`

验收标准：

- 非微信读书页面不应请求摄像头
- 微信读书阅读页应开始初始化

## 3. 首次摄像头授权与校准

1. 第一次进入微信读书阅读页
2. 允许摄像头权限
3. 保持正常阅读约 `45 秒`

验收标准：

- 页面出现“校准中”提示
- 校准结束后出现“护眼监测已开启”提示
- 后续再次进入页面时，不需要重复校准

## 4. 视觉提醒触发

1. 保持页面在前台
2. 持续阅读，不频繁眨眼
3. 等待提醒出现

验收标准：

- 页面右上角出现提醒浮层
- 同时播放轻提示音
- 不应在极短时间内连续重复提醒

## 5. 视觉提醒自动解除

1. 在提醒浮层出现后
2. 连续做 `3 次` 明显眨眼

验收标准：

- 提醒自动消失
- 不需要点击按钮

## 6. 活跃阅读态暂停

1. 打开微信读书阅读页后切换到其他标签页
2. 或在当前页保持超过 `3 分钟` 不滚动、不点击

验收标准：

- 监控暂停
- 不应继续触发提醒

## 7. 降级模式

1. 清除站点摄像头权限
2. 重新进入微信读书阅读页并拒绝摄像头

验收标准：

- 页面提示 `摄像头权限被拒绝，已切换为定时提醒模式`
- 后续仍会在活跃阅读足够久后出现提醒
- 但不依赖眨眼自动解除

## 8. 启动错误分类

1. 分别制造下面几种失败：
   - 拒绝摄像头权限
   - 让摄像头设备不可用
   - 让视觉模型资源加载失败

验收标准：

- 阅读页提示不是统一的“摄像头不可用”
- popup 能看到对应错误摘要
- options 能看到对应错误说明和建议操作

## 9. Popup 状态

1. 完成一次提醒，再打开 popup
2. 在冷却期内观察 popup
3. 等冷却期结束后再次打开 popup

验收标准：

- 可以看到当前模式
- 可以看到当前策略
- 冷却期内显示一个具体的“最早下次提醒”时间
- 冷却期结束后显示 `可立即触发`

## 10. Popup 统计

1. 完成几分钟阅读
2. 点击扩展图标

验收标准：

- 可以看到今日阅读时长
- 可以看到累计阅读时长
- 可以看到今日提醒次数
- 可以看到累计提醒次数

## 11. 策略切换

1. 打开扩展选项页
2. 依次切换 `保守 / 标准 / 敏感`
3. 刷新 options 或重新打开 popup

验收标准：

- 当前策略可以切换
- 切换后会持久化保存
- popup 会同步显示新的策略名称
- 每档都有一行简短说明

## 12. Options 导出

1. 打开扩展选项页
2. 点击 `导出按书统计 CSV`

验收标准：

- 浏览器下载一个 CSV 文件
- CSV 至少包含：
  - `date`
  - `bookTitle`
  - `readingMinutes`
  - `reminderCount`
  - `averageBlinkRatePerMinute`
  - `recoverySuccessCount`
  - `averageRecoverySeconds`

## 13. 清空数据

1. 在扩展选项页点击 `清空本地统计`

验收标准：

- popup 里的累计数据被清空
- 重新进入页面时会重新进入初始化/校准流程

## 14. 重新校准

1. 在扩展选项页点击 `重新校准`
2. 返回微信读书阅读页

验收标准：

- 历史统计仍然保留
- 扩展重新进入校准流程
- 当前模式显示回到 `视觉检测`

## 15. 回归关注点

重点观察这些风险：

- 光线过暗时是否误判
- 戴眼镜时是否能正常识别
- 侧脸或距离过远时是否持续失去人脸
- 提示音是否被浏览器静音策略拦截
- 长时间阅读后本地统计是否明显累积
