# No-Camera TTS Reminder Design

## 背景

当前扩展是围绕“摄像头 + 视觉检测 + 校准 + 眨眼提醒”设计的。  
这条路线已经和新的产品目标冲突：

- 用户明确不接受摄像头权限
- 用户希望产品行为更自由、更低权限
- 提醒应基于阅读状态和阅读时长，而不是生物特征检测
- 提醒声音希望是语音播报，而不是提示音或摄像头驱动的恢复逻辑

因此，本次不是“关闭一个开关”，而是把产品重新定义成：

`一个面向微信读书 Web 的无摄像头阅读提醒扩展`

## 目标

第一版无摄像头方案只解决一件事：

- 当用户在微信读书阅读页中持续处于活跃阅读状态时，每累计 `20 分钟` 有效阅读时间触发一次提醒

提醒表现为：

- 页面浮层
- 默认 `TTS` 语音播报固定文案

## 非目标

本次明确不做：

- 摄像头权限请求
- MediaPipe / 视觉检测
- 校准
- 眨眼率、低眨眼时段、恢复效果统计
- 普通提示音作为默认提醒方式
- 多文案轮换
- 自定义语音文案
- 微信读书以外的网站支持

## 方案比较

### 方案 1：彻底改成无摄像头提醒产品

优点：

- 权限最少
- 用户心智最简单
- 故障面最小
- 和当前用户意图完全一致

缺点：

- 会删除大量已有视觉检测相关代码和概念

这是推荐方案。

### 方案 2：关闭摄像头，但尽量保留旧视觉产品骨架

优点：

- 复用现有代码更多

缺点：

- 用户界面和产品模型会残留大量不成立的旧概念
- 后续维护复杂
- 容易重新滑回“视觉模式 / 降级模式”心智

### 方案 3：前台做无摄像头，底层保留双路线

优点：

- 以后想回头做摄像头版更方便

缺点：

- 复杂度高
- 和“彻底移除摄像头能力”的要求冲突

## 选定方案

采用 `方案 1`。

产品重新定义为一套统一模式：

- 支持页面：微信读书阅读页
- 判断用户是否正在活跃阅读
- 按累计活跃阅读时长触发提醒
- 提醒方式为浮层 + TTS

不再存在：

- 视觉检测模式
- 定时提醒模式 / 降级模式
- 校准
- 摄像头错误提示

## 核心行为

### 1. 生效范围

只在：

- `https://weread.qq.com/web/reader/*`

生效。

非支持页面：

- 不累计阅读时间
- 不触发提醒
- `popup` 中的预览提醒按钮保持禁用

### 2. 活跃阅读态

只有在下列条件同时满足时，才算“活跃阅读”：

- 当前页面是微信读书阅读页
- 当前标签页在前台
- `document.visibilityState === "visible"`
- 最近 `3 分钟` 内发生过滚动、点击、按键或滚轮操作

活跃阅读态失效时：

- 阅读时间累计暂停
- 不继续接近下一次提醒

活跃阅读态恢复时：

- 从暂停位置继续累计
- 不强制从零开始

### 3. 提醒节奏

提醒规则改成：

- 每累计 `20 分钟` 活跃阅读时间，触发一次提醒

这里的 `20 分钟` 不是自然时钟时间，而是有效阅读累计时间。

例子：

- 读 10 分钟
- 切走标签页 5 分钟
- 再读 10 分钟

此时应触发提醒，因为累计活跃阅读已达到 20 分钟。

### 4. 提醒表现

触发提醒时：

- 在当前微信读书页面右上角显示浮层
- 用浏览器 `TTS` 播放固定文案

默认固定文案：

- `请休息一下，眨眼几次，再看远处十秒。`

默认不播放普通提示音。

### 5. TTS 失败时的退化行为

如果当前浏览器环境下 `TTS` 无法成功播放：

- 仍然显示浮层
- 不自动回退为普通提示音
- 不影响后续的提醒计时

也就是说：

- 成功路径：浮层 + TTS
- 失败路径：仅浮层

### 6. Popup

`popup` 收缩为极简状态面板，只保留：

- 今日阅读时长
- 今日提醒次数
- 下次提醒时间

“下次提醒时间”的定义：

- 正在活跃阅读累计中：显示预计到达下一次提醒节点的时间
- 当前不在活跃阅读态：显示 `等待开始阅读`
- 已达到提醒点且可立即提醒：可显示 `可立即触发`

### 7. 预览提醒

现有 `预览提醒` 保留，但跟随新提醒模型：

- 仅在微信读书阅读页可用
- 点击后弹出真实提醒浮层
- 同时走当前默认提醒形式，即 `TTS`
- 不增加提醒次数
- 不改变下次提醒时间

## 数据模型

无摄像头版本只保留最小必要数据：

- 今日阅读时长
- 今日提醒次数
- 当前累计活跃阅读时长
- 下次提醒时间或等效状态

不再保留或展示：

- 眨眼基线
- 平均眨眼率
- 低眨眼时段
- 恢复成功率
- 校准状态
- 摄像头运行模式

## 需要删除的旧能力

本次设计要求明确删除以下产品概念和对应代码路径：

- 摄像头请求链路
- MediaPipe 视觉检测链路
- 页面世界视觉桥接
- 校准逻辑
- 眨眼检测
- 恢复效果统计
- `视觉检测 / 定时提醒` 双模式
- 摄像头相关错误分类

## 测试与验收重点

重点验证：

1. 非支持页面不触发提醒
2. 活跃阅读累计在页面切走后暂停
3. 累计 20 分钟后触发提醒
4. 提醒时默认走 `TTS`
5. `TTS` 失败时仍有浮层
6. `popup` 只展示极简信息
7. 旧视觉检测相关文案不再出现在 UI 和文档中

## 成功标准

满足以下条件即可认为本次产品重构成功：

- 扩展不再请求摄像头权限
- 微信读书阅读页中能稳定按累计 `20 分钟` 活跃阅读时间提醒
- 默认提醒方式变成 `TTS`
- `popup` 简化为最必要状态
- 用户不会再看到视觉检测、校准、眨眼率等旧产品概念
