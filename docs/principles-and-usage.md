# WeRead Eye Care 原理与使用说明

## 1. 这是什么

`WeRead Eye Care` 是一个面向 `Chrome + 微信读书 Web` 的护眼扩展。

它的目标不是做系统级护眼工具，而是在你用微信读书网页版阅读时：

- 判断你是否处于活跃阅读状态
- 在本地分析摄像头画面，估计你是否持续盯屏且眨眼率下降
- 用低打断方式提醒你眨眼休息
- 在本地保存阅读和提醒统计，并支持导出 CSV

V1 只支持：

- `https://weread.qq.com/web/reader/*`

不支持：

- 微信读书站点之外的网页
- 全局 macOS 级监控
- 云同步

## 2. 工作原理

### 2.1 整体流程

扩展运行时由 5 个层次组成：

1. `站点适配层`
   负责确认当前页面是不是微信读书阅读页，并提取当前书名。

2. `活跃阅读态判断`
   负责判断你是不是正在阅读。当前实现要求：
   - 标签页在前台
   - 页面可见
   - 最近 `3 分钟` 内有滚动、点击、按键或滚轮操作

3. `视觉检测层`
   在你授权摄像头后，使用本地的 `MediaPipe Face Landmarker` 模型检测人脸关键点，再基于眼部关键点计算眨眼。

4. `提醒决策层`
   在“活跃阅读态”成立时，结合持续注视时长、最近一段时间的眨眼率和个人基线，决定是否触发提醒。

5. `统计与导出层`
   把阅读时长、提醒次数、平均眨眼率、恢复效果等信息保存在 `chrome.storage.local`，并支持导出为 `CSV`。

### 2.2 如何判断“你正在阅读”

扩展不会只因为你打开了微信读书页面就一直工作。

当前实现里，只有下面条件同时满足才认为你处于阅读状态：

- 页面 URL 是微信读书阅读页
- 当前标签页在前台
- `document.visibilityState === "visible"`
- 最近 `3 分钟` 内发生过阅读相关交互

如果你切到别的标签页、页面隐藏、或长时间没有任何操作，监控会暂停。

### 2.3 如何判断“眨眼变少”

视觉模式下，扩展会做两件事：

1. `首次校准`
   首次进入微信读书阅读页并授权摄像头后，扩展会做大约 `45 秒` 的校准，记录你的自然眨眼水平。

2. `实时检测`
   后续阅读时，扩展会周期性采样摄像头画面，识别人脸关键点，并计算 `EAR (Eye Aspect Ratio, 眼睛纵横比)`。

EAR 的基本思路是：

- 眼睛睁开时，眼睑上下关键点距离较大
- 眼睛闭合时，这个距离会明显缩小
- 当 EAR 从“高于阈值”变成“低于阈值”再回到“高于阈值”，可以近似认为发生了一次眨眼

当前实现里的提醒默认规则是：

- 连续注视达到 `20 秒`
- 最近 `60 秒` 的眨眼率低于个人基线的 `60%`
- 不在提醒冷却期内

满足这些条件后，扩展就会弹出提醒。

### 2.4 提醒是怎么结束的

视觉模式下，提醒不是靠点按钮关闭，而是自动结束。

当前实现里，提醒结束条件是：

- 提醒出现后，在 `10 秒` 内识别到 `3 次眨眼`

满足后：

- 提醒浮层自动消失
- 本次恢复被记为一次成功恢复
- 记录恢复耗时

### 2.5 没有摄像头时怎么办

如果发生下面任一情况：

- 你拒绝摄像头权限
- 浏览器拿不到摄像头
- 视觉模型初始化失败

扩展会自动进入 `降级模式`。

降级模式下：

- 不再进行眨眼检测
- 只按活跃阅读时间触发提醒
- 当前默认是每 `20 分钟` 提醒一次
- 统计仍然保留，但没有眨眼率数据

## 3. 隐私与数据边界

这是当前实现里最重要的约束之一。

### 3.1 不会做什么

扩展不会：

- 上传摄像头原始画面
- 保存视频文件
- 保存截图
- 把阅读统计发到服务器
- 把数据同步到云端

### 3.2 会保存什么

扩展默认把数据保存在当前 Chrome 用户的本地扩展存储里，也就是 `chrome.storage.local`。

当前保存的数据包括：

- 按天的阅读时长
- 按天的提醒次数
- 按天的平均眨眼率
- 低眨眼时段累计时长
- 恢复成功次数
- 平均恢复耗时
- 按书名聚合的统计明细
- 个人眨眼基线
- 当前运行模式：`vision` 或 `fallback`

### 3.3 导出文件里包含什么

当前导出为 `CSV`，包含按书聚合明细，至少有这些列：

- `date`
- `bookTitle`
- `readingMinutes`
- `reminderCount`
- `averageBlinkRatePerMinute`
- `recoverySuccessCount`
- `averageRecoverySeconds`

## 4. 如何安装

### 4.1 本地构建

在项目根目录执行：

```bash
npm install
npm test -- --run
npx tsc --noEmit
npm run build
```

构建完成后，会生成：

- `dist/`

### 4.2 加载到 Chrome

1. 打开 `chrome://extensions`
2. 打开右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择当前项目里的 `dist/` 目录

加载后，你会看到扩展图标和扩展详情。

## 5. 如何使用

### 5.1 第一次使用

1. 在 Chrome 中打开微信读书阅读页  
   例如：`https://weread.qq.com/web/reader/...`

2. 扩展检测到支持页面后会开始初始化

3. 如果浏览器弹出摄像头权限请求，点击允许

4. 首次允许后，扩展会进行约 `45 秒` 的本地校准  
   这段时间尽量保持自然阅读，不要刻意频繁眨眼

5. 校准完成后，扩展进入视觉检测模式

### 5.2 日常使用

使用时不需要额外操作。

只要你满足下面条件，扩展就会自动工作：

- 正在微信读书阅读页
- 页面在前台
- 最近有阅读交互

当检测到你连续盯屏且眨眼偏少时，页面右上角会出现提醒浮层，并播放轻提示音。

### 5.3 提醒后怎么恢复

视觉模式下：

- 扩展会等待你完成 `3 次眨眼`
- 检测到后，提醒自动关闭

降级模式下：

- 提醒会显示一段时间后自动消失
- 不要求摄像头参与确认

### 5.4 查看统计

点击扩展图标，可以看到：

- 今日阅读时长
- 累计阅读时长
- 今日提醒次数
- 累计提醒次数
- 恢复成功率摘要

### 5.5 导出统计

当前版本的导出入口在扩展的 `Options Page`：

1. 打开 `chrome://extensions`
2. 找到 `WeRead Eye Care`
3. 点击 `扩展程序选项`
4. 点击 `导出按书统计 CSV`

### 5.6 清空本地统计

同样在 `Options Page` 中：

1. 打开扩展选项页
2. 点击 `清空本地统计`

这会清除当前保存在本地扩展存储里的统计和校准状态。

## 6. 当前默认参数

当前代码里的默认参数是：

- 活跃阅读超时：`3 分钟`
- 视觉采样间隔：`750 ms`
- 统计采样间隔：`5 秒`
- 首次校准时长：`45 秒`
- 连续注视阈值：`20 秒`
- 低眨眼判定窗口：`60 秒`
- 低眨眼阈值：低于基线的 `60%`
- 提醒冷却时间：`3 分钟`
- 连续眨眼恢复窗口：`10 秒`
- 恢复条件：`3 次眨眼`
- 降级模式提醒间隔：`20 分钟`

## 7. 你需要知道的限制

当前版本仍然是 MVP，有这些边界：

- 只支持微信读书阅读页，不支持其他网页
- 不支持系统级全局提醒
- 当前没有多设备同步
- 当前没有复杂设置项界面
- 眨眼检测依赖前置摄像头、人脸角度和光照条件
- 戴眼镜、低光、侧脸、躺着阅读时，误报和漏报概率会升高

## 8. 常见问题

### 8.1 为什么没有弹摄像头权限

常见原因：

- 当前页面不是微信读书阅读页
- 扩展没有正确加载 `dist/`
- 你之前已经拒绝过权限，需要到站点权限里重新允许

### 8.2 为什么一直不提醒

常见原因：

- 页面不在前台
- 你长时间没有滚动/点击，扩展认为你不在活跃阅读状态
- 环境光太差，导致人脸关键点识别不稳定
- 首次校准不稳定，基线偏高或偏低

### 8.3 为什么提醒太频繁

可能原因：

- 当前光照或摄像头角度导致眨眼识别偏少
- 你离摄像头太远
- 当前默认阈值还没有针对你的习惯调优
