# WeRead Eye Care 原理与使用说明

## 1. 这是什么

`WeRead Eye Care` 是一个面向 `Chrome + 微信读书 Web` 的阅读提醒扩展。

当前版本的产品定义很简单：

- 只按你的活跃阅读时间提醒休息

V1 只支持：

- `https://weread.qq.com/web/reader/*`

不支持：

- 微信读书之外的网页
- 全局 macOS 级提醒
- 云同步

## 2. 工作原理

### 2.1 整体流程

扩展运行时由 4 个部分组成：

1. `站点识别`
   只在微信读书阅读页启动内容脚本。

2. `活跃阅读态判断`
   判断你当前是不是在真正阅读，而不是只是把页面挂在后台。

3. `提醒调度`
   按累计的活跃阅读时间触发提醒，当前默认是每 `20 分钟` 一次。

4. `提醒展示`
   在页面里显示全屏遮罩，并播放扩展内置固定语音。

### 2.2 什么叫“活跃阅读”

扩展不会只因为页面打开着就持续累计时间。

只有下面条件同时成立时，才认为你处于活跃阅读状态：

- 页面 URL 是微信读书阅读页
- 当前标签页在前台
- `document.visibilityState === "visible"`
- 最近 `3 分钟` 内发生过滚动、点击、按键或滚轮操作

这意味着：

- 切到别的标签页时会暂停
- 页面隐藏时会暂停
- 长时间没有任何阅读操作时会暂停

恢复操作后，会从之前累计到的位置继续，而不是清零。

### 2.3 提醒是怎么触发的

当前提醒不是按自然时间，而是按“有效阅读累计时间”。

默认规则是：

- 每累计 `20 分钟` 活跃阅读时间，触发一次提醒

举例：

- 先读 `10 分钟`
- 切走标签页 `5 分钟`
- 再读 `10 分钟`

这时会在第二段阅读结束后触发提醒，因为只累计了 `20 分钟` 的有效阅读时间。

### 2.4 提醒长什么样

提醒触发后会发生两件事：

1. 微信读书页面出现全屏提醒遮罩
2. 扩展播放内置固定语音

当前默认文案是：

`请休息一下，眨眼几次，再看远处十秒。`

这个遮罩会阻止底层页面继续交互。

你必须手动点击：

- `我知道了`

提醒才会关闭。

### 2.5 如果固定音频播放失败怎么办

当前版本把扩展内置音频当作默认提醒方式。

如果浏览器当前不能正常播放这段音频：

- 页面全屏提醒仍然会出现
- 不会自动退回到系统 `TTS`
- 下一轮提醒计时不受影响

### 2.6 Popup 和 Options 会显示什么

`popup` 显示：

- 今日阅读时长
- 今日提醒次数
- 阅读状态
- 下次提醒
- `预览提醒` 按钮

其中：

- 活跃阅读中会显示 `计时中 · X 分钟`
- 当前不在活跃阅读态时显示 `已暂停 · X 分钟`
- 活跃阅读中会显示预计达到下一个提醒节点的时间
- 当前不在活跃阅读态时显示 `等待开始阅读`
- 已经达到提醒点时显示 `可立即触发`

`options` 只保留最小状态页：

- 今日阅读时长
- 今日提醒次数
- 阅读状态
- 下次提醒
- 导出 CSV
- 清空本地统计按钮

### 2.7 如何预览提醒样式

如果你只想确认提醒会怎么显示，不需要等满 `20 分钟`。

当前实现里：

- 当活动标签页是微信读书阅读页时，`popup` 会启用 `预览提醒`
- 点击后，会在当前阅读页里弹出真实提醒遮罩
- 同时会播放当前默认的内置固定语音
- 预览不会计入提醒次数，也不会改变下次提醒时间
- 预览也必须手动点击 `我知道了` 才会关闭

如果当前活动标签页不是微信读书阅读页，按钮会置灰，并显示：

- `仅在微信读书阅读页可预览`

## 3. 隐私与数据边界

### 3.1 不会做什么

扩展不会：

- 上传阅读数据
- 把数据同步到服务器
- 保存原始页面内容
- 保存截图或录音

### 3.2 会保存什么

扩展默认把数据保存在当前 Chrome 用户的本地扩展存储里，也就是 `chrome.storage.local`。

当前保存的数据只有：

- 按天的阅读时长
- 按天的提醒次数
- 当前累计了多少活跃阅读时间
- 当前是否处于活跃阅读态
- 下次提醒时间

## 4. 如何安装

### 4.1 本地构建

在项目根目录执行：

```bash
npm install
npm test -- --run
npx tsc --noEmit
npm run build
```

构建完成后会生成：

- `dist/`

### 4.2 加载到 Chrome

1. 打开 `chrome://extensions`
2. 开启右上角 `开发者模式`
3. 点击 `加载已解压的扩展程序`
4. 选择当前项目里的 `dist/`

## 5. 如何使用

### 5.1 日常使用

1. 打开微信读书阅读页
2. 正常阅读
3. 扩展会自动累计活跃阅读时间
4. 到达 `20 分钟` 后触发提醒

使用过程中不需要额外授权。

### 5.2 查看状态

点击扩展图标，可以看到：

- 今日阅读时长
- 今日提醒次数
- 阅读状态
- 下次提醒

### 5.3 清空本地统计

在扩展的 `Options Page` 里点击：

- `清空本地统计`

这会清空本地扩展存储里的统计和当前累计状态。

### 5.4 导出 CSV

在扩展的 `Options Page` 里点击：

- `导出 CSV`

当前导出的字段只有 4 列：

- `date`
- `bookTitle`
- `readingMinutes`
- `reminderCount`

这和当前版本实际保存的数据保持一致。

## 6. 当前默认参数

当前代码里的默认参数是：

- 活跃阅读超时：`3 分钟`
- 统计采样间隔：`5 秒`
- 提醒间隔：`20 分钟`

## 7. 你需要知道的限制

当前版本仍然是 MVP，有这些边界：

- 只支持微信读书阅读页，不支持其他网页
- 不支持系统级全局提醒
- 当前没有多设备同步
- 当前没有自定义提醒音频
- 当前没有多种提醒节奏

## 8. 常见问题

### 8.1 为什么没有提醒

常见原因：

- 页面不在前台
- 页面不可见
- 最近 `3 分钟` 没有任何阅读操作，扩展暂停累计
- 实际累计到的活跃阅读时间还不到 `20 分钟`

### 8.2 为什么 popup 里显示“已暂停 · X 分钟”

这表示当前不在活跃阅读态。

通常是下面几种情况之一：

- 当前标签页不是微信读书阅读页
- 页面在后台
- 页面虽然打开着，但已经超过 `3 分钟` 没有交互

`下次提醒` 这一栏在这种情况下仍会显示：

- `等待开始阅读`

### 8.3 为什么页面有浮层但没有语音

说明提醒已经触发，但浏览器当前没有正常播放扩展内置音频。

当前实现下：

- 不会自动改成系统 `TTS`
- 仍然会显示全屏提醒

优先检查浏览器对页面音频播放的支持情况，再重试。

### 8.4 如何确认固定音频有没有成功触发

每次提醒播放后，扩展都会把当前固定音频的结果写到页面根节点的 `dataset` 里，方便手工验收。

你可以在微信读书页面的 DevTools Console 里查看：

```js
document.documentElement.dataset.wereadEyeCareReminderAudioPath
document.documentElement.dataset.wereadEyeCareReminderAudioStatus
document.documentElement.dataset.wereadEyeCareReminderAudioErrorMessage
```

如果你想继续开发，主要入口在这里：
